# Release notes v1.4.0

- 修复了当跳过下载模组时，手动安装模组后，已跳过模组列表不会更新的问题
- 跳过下载模组的警告窗口现在可以被收起
- 修复启动器日志可能因下载事件重复记录完整安装状态而快速膨胀的问题
- 启动器日志现按 10 MiB 自动轮转并最多保留 5 个文件
- Modrinth 请求日志保留官方或镜像目标、来源、重试次数和脱敏 URL，不再记录查询参数中的临时签名
- 错误日志导出改为流式压缩，降低大日志导出时的内存占用
- WARN 和 ERROR 日志在 30 MiB 边界内保持完整，超过边界前自动轮转且不拆分单个事件
- 启动器日志创建超过三天后自动删除
- 修复 Fabric/Modrinth 实例内容页手动下载状态 watcher 重复写入相同 `Map`、触发递归更新并持续加载的问题
- 大幅加强 Java 自动检测：新增自定义目录关键词广度搜索（深度上限 5、带目录数量保险丝）、全平台 JAVA_HOME 及其同级目录检测、官方启动器自带运行时检测（含微软商店版），并补充 Azul Zulu 与 BellSoft Liberica 的注册表项
- 修复 Windows 注册表 Java 检测中部分键路径反斜杠转义错误导致检测从未生效的问题，并正确读取 Eclipse Adoptium 的嵌套安装路径
- Java 检测结果现在会持久化缓存，打开 Java 选择窗口即时显示；后台以防抖方式自动重扫，发现变化时实时刷新列表
- Java 检测的各收集来源现在并行执行，冷启动首次扫描耗时显著降低
- 启动实例缺少所需 Java 版本时，优先复用系统中已检测到的同版本 Java，而不是直接下载新的运行时
